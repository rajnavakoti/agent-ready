How to fill the gaps

Demand-Driven Context (DDC) is a methodology for building domain knowledge bases through agent failure. Like TDD — but for knowledge.

01

Give the agent a real problem

An incident to diagnose, an integration to design, a process to explain.

02

Let it fail

The agent identifies exactly what knowledge is missing. That's the demand signal.

03

Curate the minimum context

A domain expert fills only the gaps. Structured, connected, machine-readable.

04

Agent succeeds. Knowledge compounds.

Each cycle builds reusable context. After 20-30 problems, the agent knows your domain.
